By the end of our first few Node.js classes, we had to go through some Js basics. We worked with arrays, objects, strings and did some console-logging as well. Here you can see a few code snippets from our first classes:


Once the basics were covered and we were able to read the written code, we started to create our own servers. This is the structure of the server, and to create it I will explain the process in steps...

STEP 1: Create empty folder and make a "package.json" file

STEP 2: Open the "package.json" file and on line one, write curly brackets (as an annotation for an empty object, "{}")

STEP 3: Open up the terminal and do a "cd" command (to change directory) and specify path where you have created the "package.json" file

STEP 4: Run "npm install express" or "npm i express" in the terminal - this will install the EXPRESS package which we will use as our minimalistic web framework for Node

After these steps, 2 things happend - the Express dependency has been added to the "package.json" file, inside the empty curly-brackets annotation and a "package-lock.json" file was created, as well as a "node_modules" folder. This folder which is automatically added to the ".gitignore" file which acts as cache for all of the external modules we add to our project. To finish off the structure, we create an "app.js" Javascript file in which we will write code to use the Express package and create a CRUD REST API as well as a Web Server for serving HTML pages like in this case.

Setting up app.js

Here we see an "app.js" file in which we start to use the Express package. First we import the package by using the require method and setting it to a const called "express" - this is similar to the Java "import" statement. Then we create an "app" const and assign the "express" package to it by calling "express()". Now the "app" const contains all of the Express methods and functions we will use. First one is the GET method which actually represents the HTTP GET method - used for requesting and reading data from a specified source.

When calling this "get()" method, we pass the following params:


1. PATH / ENDPOINT:
- this represents the endpoint through which we will access the data being read
- example "localhost:8080/" where "/" is the endpoint, and it servers an empty JSON object

2. REQ & RES:
- "req" as in request, mostly used when a path variable or a query string is passed in the URL
- "res" as in response, used to send/serve the actual data
- "/me" endpoint - response object sends an object called "me", and this JSON object is represented on this exact path

Last line of code is the "app.listen()" method, which is used to assign a port through which we will access any of the endpoints defined in the "app.js" - any free port number can be passed here as so -- "app.listen(8081)" which leads to "localhost:8081" and then any of the endpoints, either through the browser or for example Postman. This method later becomes more complex, as we create a callback function for error-logging and we also play around with scripts and running our server on custom ports.
